## 1. Why（动机）

LangFix 当前把**用户母语写死为中文**、**目标（被纠错）语言假设为英文**：
- Prompt 硬编码「用户母语是中文」「用【中文】逐条解释」（`Prompt.swift`）；
- 结构化字段 `reason_zh` / `summary_zh` / `translation_zh` 及整套 UI 文案全写死中文（18+ 处硬编码）；
- 多语言混排规则是「**只修目标语言片段、不翻译其余语言**」（living spec R「多语言混排只修目标语言」）；
- 无语言配置、无首启引导。

用户需求（RAS-53 讨论定稿）：
1. **可配语言**：目标语言（预期纠正的语言，`{中文, 英文}` 二选一）+ 用户语言（母语，驱动 UI 展示与解释/翻译）。
2. **首启引导**：首次使用先强制打开设置页配语言，之后可改。
3. **混排统一到目标语言**（行为反转）：如「英文里嵌中文」应**统一转写为目标语言表达**，而非保留不译。
4. **追问放宽**：追问可问任意语法/语言问题（放宽话题范围），但**不改主结果的护栏保持不变**。

| grammar-review | MODIFIED | 「多语言混排」：由「只修目标语言、不翻译其余」**反转**为「非目标语言片段统一转写为目标语言」（护栏流程不变） |

**新增 capability**：无。

## 3. Out of Scope（明确不做）

- **不扩目标语言集**：目标语言 V1 仅 {中文, 英文}。
- **不做超出 {中,英} 的 UI 语言**：UI/解释语言 V1 支持 {中文, 英文}；非中英 locale 按 §5 truth table 回退英文。
- **不弱化任何红线**：Constraint-1/2/3/4 不动；最小改动护栏的 strict-retry 流程不变、不默认关闭；追问「不改主结果」护栏保留。
- **不做「追问驱动重纠错 / 翻译整段新文本」**：追问放宽后仍不得输出可作整段替换结果的全文；需对新文本纠错/翻译请走一次新的 review 链路（见 spec）。
- **不改窗口自适应/字号**（字号属 RAS-54）。
- **不定交互形态与实现**：设置控件、首启 UX、字段兼容具体写法、i18n 基建选型（String Catalog 等）归技术方案阶段。
- **不写代码**：本阶段只产出 proposal + spec-delta。

## 4. Success Metrics（成功指标）

- **语言可配且约束成立**：可分别配目标/用户语言；**目标语言恒 ∈{中,英} 且不等于用户语言**（确定性可测）。
- **首启强制配语言（新装）/ 老用户不被打断**：新装未配 → 触发前强制配语言；老用户升级 → 自动迁移为已配置、不强制进设置（确定性可测）。
- **混排统一到目标语言**：目标=英输入「英文夹中文」→ 夹带片段转写为英文；目标=中输入「中文夹英文」→ 夹带片段转写为中文；目标语言片段仍最小改动（确定性可测）。
- **护栏流程不变**：混排导致 editRatio 超阈时，**仍先 strict retry 一次、仍超才提示「改动较大」并出结果**，不跳过重试、不关护栏（确定性可测）。
- **解释语言随用户语言**：用户语言=英时解释/总评/直译为英文（确定性可测）；旧 `_zh` 数据仍能解析、缺字段 fail loud。
- **追问放宽但护栏在**：能解答与本次结果无关的通用语法问题；同时「贴新文本/让它给一版替代全文」仍被输出护栏拦截、主结果不变（确定性可测）。
- **UI 随用户语言**：{中,英} 两态 UI 文案随用户语言切换（确定性可测）。

## 5. Clarifications（用户已拍板，含确定性 truth table）

### 已定稿
| # | 决策 |
|---|---|
| 中文取代 | 用户语言**直接取代**写死中文——`reason_zh`/`summary_zh`/`translation_zh`/整套 UI 全改为用户语言驱动 |
| 首启引导 | 新装/未配语言先强制开设置配语言，之后可改；老用户升级自动迁移为已配置 |
| 追问放宽 | 护栏保留（不改主结果/不吐替代全文/易失内存/注入防御），仅放宽追问**话题范围**；基础纠错照常 |
| 混排 | 「英文嵌中文→统一转目标语言表达」= 按目标语言纠正；与目标语言配置对齐 |
| 混排 × 护栏 | 语言统一仍**完整走原最小改动护栏流程**：算 editRatio → 超阈**先 strict retry 一次** → strict 后仍超才展示「改动较大」并**始终出结果**。Constraint-3 算法与重试不变、不绕过、不默认关闭。混排比例偏高属预期，可提示 |

### 语言默认 truth table（确定性，无歧义）
| 系统 locale | 用户语言 | 目标语言 | 触发 |
|---|---|---|---|
| `zh*` | 中文 | 英文 | 新装：预填并强制首启确认 |
| `en*` | 英文 | 中文 | 新装：预填并强制首启确认 |
| 其他（ja/de/…） | 英文 | 中文 | 新装：预填并强制首启确认 |
| 老用户升级 | 中文 | 英文 | 自动迁移为已配置，不强制首启（等价现状行为，可后续手改） |

> 目标语言恒取 {中,英} 中异于用户语言者，上表各行均满足「目标≠用户」。首启为**强制**配置，故非中英 locale 的「英/中」默认仅是预填起点。
> 🟡 **仅一处留否决**：非中英 locale 默认「用户语言=英文、目标=中文」——若你希望非中英 locale 走别的默认（如按其原生 locale 显示 UI，则 UI 语言集需超出 {中,英}、i18n 面变大），一句话告知即可调整；不回复则按上表推进。

## 6. 迁移（老用户）—— 无冲突路径

- **老用户升级**（已装、无语言配置键）：升级时**自动写入** 用户语言=中文、目标语言=英文，并**标记为已配置**——等价现状行为，**不触发强制首启**（避免打断老用户）。
- **新装**：无配置 → 首次触发前强制进设置配语言（按 truth table 预填）。
- **字段兼容**（升为 Requirement，非仅迁移说明）：新语言中立字段（`reason`/`summary`/`translation`）优先；模型历史返回或回归样例中的旧 `_zh` 字段**兼容读取**；关键解释字段缺失时 **fail loud**，不静默丢 explanation/summary/translation。

## 8. 风险

| 风险 | 可能性 | 影响 | 缓解 |
|---|---|---|---|
| 🔴 混排反转被实现方误读为「可跳过 strict retry / 关护栏」 | 中 | 高（红线） | §5 明确：护栏流程与 strict retry 完全不变、不绕过；spec 场景断言重试仍触发 |
| 🔴 追问放宽后用户贴新文本要 AI「给一版全文/翻译整段」绕过 review 链路 | 中 | 高（红线） | Out of Scope + spec：追问可解释/给短例句，但不得输出可作整段替换的全文；新文本走新 review |
| 混排统一误伤目标语言片段（过度改写） | 中 | 中 | prompt 仅对非目标语言片段转写、目标语言片段仍最小改动；护栏比例仍生效；补两方向场景 |
| 目标语言=用户语言导致自我纠错悖论 | 中 | 中 | 硬校验「目标≠用户」，truth table 各行成立，UI 不允许配相同 |
| 字段去后缀破坏旧数据/回归 | 中 | 中 | 升为 Requirement：新字段优先 + 旧 `_zh` 兼容读 + 缺字段 fail loud；既有测试须全绿 |
| 混排翻译放大「AI 回显 original 被污染」风险 | 中 | 中 | 重申 living spec「修正基准一致性」：diff/editRatio 恒以本地真实输入为 original |
| UI 本地化面过大拖慢交付 | 中 | 中 | V1 UI 语言集限 {中,英}；非中英回退英文 |
| 与 RAS-54 并行冲突 | 中 | 中 | §7 串行：RAS-54 先合、本 change 后 rebase |
