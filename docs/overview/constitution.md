# LangFix Constitution

> **Owner**: n374
> **创建日期**: 2026-06-29
> **最后更新**: 2026-06-29
> **修改原则**: 红线"只增不减"——新增红线属正常演进；移除/弱化红线必须经过专门的 change 与 Owner 同意。

## 1. 治理原则

1. **最小改动优先**：LangFix 是「纠错器」不是「润色器」。任何让输出偏离用户原意/原语气的改动都需被约束与可见。
2. **隐私默认**：用户输入的文本是敏感数据，默认不持久化、不外泄到日志。
3. **可信优于花哨**：宁可不做流式、不做自动替换，也要保证结果可信、可核对、可撤销（复制而非改原文）。

## 2. 红线 / Hard Constraints

> 任何 change 触及以下红线必须停下，按 doc-init skill §11 红线检查机制处理。

### Constraint-1: API key 只进 Keychain
- **内容**: API key 仅存储于 macOS Keychain；绝不写入 UserDefaults、plist、日志、PopClip 扩展配置或任何明文文件。
- **理由**: 密钥泄露直接造成账单/数据风险；明文存储不可接受。
- **触发示例**: 把 key 存进 `UserDefaults`、打印到 log、写进配置文件、随 crash report 上报。
- **强制覆盖路径**: 在 design.md Clarifications 写完整覆盖记录（审批人 / 日期 / 原因 / 影响 / 回滚 / 验收）。

### Constraint-2: 不记录消息内容
- **内容**: 默认不把用户原文（`original`）与修正文（`corrected`）写入任何日志/文件/历史。日志只允许：request id、耗时、token 计数、HTTP 状态码、错误类型。
- **理由**: 用户输入文本敏感；落盘即风险面。
- **触发示例**: debug 日志打印整段文本；持久化「历史记录」默认开启；崩溃上报携带文本。
- **强制覆盖路径**: 同上；若要做「历史记录」功能，必须默认关闭 + 显式开关 + 本地加密存储，并在 ADR 记录。
- **数据流向声明（必须明示，勿误读为本地处理）**: 「不落盘/不记录」**不等于不外发**——选中文本会通过 HTTPS 发送到**用户自己配置的 OpenAI 兼容端点**做处理。设置页与隐私说明必须明确写出这一点；敏感内容用户应自行选择可信端点或本地模型。详见 spec NFR-5。

### Constraint-3: 最小改动护栏不可被破坏
- **内容**: 主修正结果（`corrected`）以「对 `original` 的最小改动」为目标。改动比例护栏（见 [ADR-0004](../decisions/0004-minimal-edit-guard.md)）**不得被删除或默认关闭**。护栏语义是**约束+提示而非阻断**：超阈值时重试并强提示，**始终展示**一个结果（不卡住用户）；「更地道整体改写」只能作为**单独标注的可选项**，不得替代主结果。
- **注**: 「最小改动」是被护栏保护的*目标与提示*，不是「必须低于阈值否则不出结果」的硬门——硬门会让用户在 AI 偶发过度改写时一无所得，反而更糟。
- **理由**: 整段重写改变原意/语气是用户最反感、最不敢用的失败模式，是产品的核心价值红线。
- **触发示例**: 默认展示「整体润色版」作为主结果；移除 diff 比例护栏；prompt 放开「可自由改写」。
- **强制覆盖路径**: 同上。

### Constraint-4: 不擅自修改用户原选区（V1）
- **内容**: V1 不通过 Accessibility/paste-back 自动替换用户原文，只提供「复制修正结果」。
- **理由**: 自动改原文一旦误操作不可逆，破坏信任。
- **触发示例**: 默认开启「自动替换选区」。
- **强制覆盖路径**: 若后续做「替换选区」必须为显式、非默认、可撤销，并经 ADR。
