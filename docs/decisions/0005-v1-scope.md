# ADR-0005: V1 范围——非流式、不自动替换选区

- **状态**: Accepted
- **日期**: 2026-06-29
- **Owner（决策者）**: n374
- **Reviewers**: n374
- **关联 change**: —
- **影响 capability**: grammar-review

## 1. 上下文

为让 V1 尽快可用且可信，明确砍掉两个「看起来很酷但风险/复杂度高」的特性：流式输出、自动替换用户原选区。

## 2. 决策

V1 **不做流式**、**不自动替换原选区**：
- AI 调用非流式，等待期显示可取消 loading。
- 只提供「复制修正结果」，不通过 Accessibility/paste-back 改写源应用里的选区。

## 3. 理由

- **非流式**：目标文本通常较短；结构化 JSON 的流式增量解析复杂度高、收益低；非流式 + loading 更稳更可控。
- **不替换选区**：自动改原文需 Accessibility 权限或 paste-back 模拟，误操作不可逆，破坏信任（红线 Constraint-4）。先把「复制」做到 100% 可靠。

## 4. 后果

- **正面**: 实现简单、结果可信、无不可逆操作。
- **负面**: 体验上少了「动态感」和「一键替换」的便利。
- **中立**: 二者均为后续可选演进（流式可加 `input_json_delta` 增量解析；替换可做成显式、非默认、可撤销）。

## 5. 备选方案

| 方案 | 优点 | 缺点 | 为什么不选 |
|---|---|---|---|
| V1 就上流式 | 反馈更即时 | 结构化流式解析复杂、易错 | 收益不抵复杂度，延后 |
| V1 就做自动替换选区 | 省一次粘贴 | 需 Accessibility、误操作不可逆 | 违背可信/可撤销原则（红线） |

## 6. 实施

- 验收：spec R5（可取消 loading）、R10（不改原选区）。
- 后续若放开任一项，需新开 ADR 并评估对红线 Constraint-4 的影响。
